Contributing to Yarel
=========================================

Please, follow this guide abot how contributing to the project:
- Clearly identify the feature to implement or the bug to solve
- Create a branch. Its name  *must be informative*. 
For example, if it is a sub-branch of develop, use the format develop-<name>, 
where <name> identifies the feature or bug the branche is related to
- Create an issue.  Its name  *must be informative*. 
It must describe a feature or a bug, saying why it is an issue and describing its implementation 
(the supplied templates are a possible guide, though not mandatory)
- Implement the solution in the appropriate branch.
If the issue can be broken into sub-issues, create sub-branches and sub-issues accordingly.
- Create a *pull request* once the implementation is complete.
A pull request is necesasary for every branch and sub-bnranch.
In absence of objections merge.
